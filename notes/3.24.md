## bottleneck in code: GEPA optimizing adversary: 
sequential calls in inner loop of coevolution

REINFORCE (× iters_per_stage):
  orig_prompt → [ADV] → rewrite → [TARGET] → resp → reward/judge → update ADV weights
  (× batch_size per iter)

GEPA attacker (≤ max_metric_calls):
  candidate_attacker + input → [ADV] → rewrite → [TARGET] fixed_defense → resp → score → reflection

GEPA defender (≤ max_metric_calls):
  fixed_attacker + input → [ADV] → rewrite → [TARGET] candidate_defense → resp → score → reflection

Eval:
  per example: orig → [ADV] → rewrite → [TARGET] (pooled) → … → batch judge
